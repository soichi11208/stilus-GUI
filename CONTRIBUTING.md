# コントリビューションガイド

stilus への貢献に興味を持っていただきありがとうございます！
以下にざっと目を通してから、進んでください。

## 設計思想

stilus は以下の原則により設計されるぅ。

- **依存性の排除**: ライブラリへの依存を最小限にし、可能な限りベンダリング、自前実装を行うことで、ディストロに依存せず動作することを目指している。
- **完全静的リンク**: musl libc と Zig ツールチェーンを組み合わせ、ランタイムに動的ライブラリを一切必要としないバイナリを生成!!
- **シンプル**: 重厚なフレームワークではなく、低レイヤーの描画機能と軽量なウィジェットツリーを提供する。
- **Linux 専用**: 当面の間、Windows や macOS への対応は予定していません。しかしまぁしたくないわけではない。

## 開発環境の構築

開発環境は**当然ながら**動作環境たるLinuxを想定している。

僕はDebian13を使用してるが、それである必要は特にない。

### 必要なツール

- **Zig (v0.16.0 以上)**: musl 向けクロスコンパイラとして使用します。
  - [ziglang.org](https://ziglang.org/download/) からバイナリをダウンロードして PATH に追加してください。
- **Meson**: ビルド定義システム。
  - `pip install meson` または OS のパッケージマネージャ（`apt install meson` 等）でインストールしてください。
- **Ninja**: 高速なビルドツール。
  - `apt install ninja-build` 等でインストールしてください。

Debian系じゃないディストロは...まぁggrks

### ビルド手順

デフォルトでは musl による完全静的ビルドを基本として想定する。

```sh
export PATH="$PWD/tools:$PATH"
meson setup build-musl --cross-file cross-musl.txt --buildtype=release
meson compile -C build-musl
```

## テストと検証

### テストの実行

```sh
meson test -C build-musl --print-errorlogs
```

※ `test_x11_integration` は実際のディスプレイが必要です。CI 環境やヘッドレス環境では `xvfb-run` 経由で実行してください。

### ゴールデンイメージテストについて

描画ロジックを変更した場合、参照画像（`test/golden/*.ppm`）を更新する必要があります。

1. `--regen` フラグを付けてテストを実行し、画像を再生成します。
2. **重要**: 再生成された画像が意図通りであるか、必ず目視で確認してください。
3. 確認後、生成された画像をコミットに含めてください。

## コントリビューションのルール

### ビルドを壊さない

**ビルドが通らない状態や、既存のテストが失敗する状態でコードをプッシュ・プルリクエストしないで。**
変更を加えた後は、必ず `meson compile` と `meson test` が正常に完了することを確認してね。

### 依存関係の追加

- **Copyleft (GPL/LGPL/MPL等) のライブラリは原則として導入しないでください。**
- MIT, BSD, Apache-2.0, Public Domain などの許容的なライセンスのライブラリのみ受け入れる。

## コーディング規約

以下のスタイルを推奨。

- **言語標準**: C++20。
- **命名規則**:
  - **クラス・構造体**: `PascalCase` (例: `Window`, `WidgetTree`)
  - **関数・メソッド**: `snake_case` (例: `request_redraw()`, `set_root()`)
  - **メンバ変数**: `snake_case_` (末尾にアンダースコアを付ける。例: `impl_`, `root_`)
  - **ローカル変数**: `snake_case` (例: `cur`, `next`)
- **フォーマット**:
  - インデントはスペース 4 つ。
  - 中括弧 `{` は行末に配置（K&R / Egyptian style）。
- **設計パターン**:
  - 内部実装を隠蔽したい場合は `detail` 名前空間や Pimpl パターン（`Impl` 構造体）を活用。

## Vibe Coding

- 特に禁止はしませんし、僕もつかってる。

- ただし再三にはなるがビルドできない状態で放置しない。

- 学習データに使用される設定でも問題はない。

- ただし古かったり小さかったりして正常な動作に支障がある、Coding規約にあまりにも準拠できないといったことは避けてください。その場合の責任を取るのはAnthropicでもOpenAIでもMoonshotAIでもZaiでもAlibabaでもDeepMindでもなくあなたです。
